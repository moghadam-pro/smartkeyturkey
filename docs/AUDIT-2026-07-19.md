# WordPress infrastructure audit — 2026-07-19

Audit mode: authenticated read-only review of the production WordPress dashboard. No settings, plugins, content, themes, or files were changed.

## Executive status

**Overall: RED — production development must not start yet.**

The base server resources are adequate for the planned site, but there is no backup or staging environment, WordPress memory is below WPML's minimum, required WPML String Translation is missing, and the installed premium packages do not have verifiable official provenance/update paths.

## Red findings — resolve before implementation

### No backup or staging

- The site is explicitly configured as `production`.
- No backup or tested restore path has been provided.
- No staging environment exists.

**Required action:** create a full files/database backup, verify that it can be restored, then create a password-protected and noindex staging copy. Do not develop or bulk-import on production.

### Premium plugin provenance and update integrity

The dashboard identifies these active premium packages as distributed by `20script`, rather than their official vendors:

- Elementor Pro 4.1.2
- Rank Math SEO PRO 3.0.107
- WPML Multilingual CMS 4.6.13

Elementor Pro also reports an available update but no automatic update path. WPML is unregistered.

**Required action:** obtain official licenses, replace the packages with clean official downloads, register them, and freeze tested versions on staging. Treat the current third-party packages as untrusted until replaced. Never enter license keys into GitHub or Slack.

### WordPress memory below WPML minimum

- PHP memory limit: 512 MB
- WordPress front-end memory limit: 40 MB
- WordPress admin maximum memory: 512 MB

**Required action:** set `WP_MEMORY_LIMIT` to 256 MB on staging and production after confirming available server memory.

### Required WPML component missing

WPML and Rank Math both report that WPML String Translation is not installed/active.

**Required action:** install the official String Translation package from the licensed WPML account on staging, then test Elementor templates, forms, Rank Math strings, custom post types, fields and taxonomies.

## Recommended architecture decision

- Keep Hello Elementor as the maintained parent theme.
- Create a version-controlled `smartkey` child theme for brand tokens, fonts, dark/light behavior, RTL-safe CSS and small integration hooks.
- Create a separate version-controlled `smartkey-core` plugin for custom post types, taxonomies, fields, importers, business rules, WPML configuration and Elementor integrations.
- Keep page composition and Theme Builder templates in Elementor Pro, with exports tracked as safe artifacts where practical.

## Safe remediation sequence

1. Obtain hosting/control-panel access and identify its native snapshot/staging features.

3. Create protected staging and disable indexing/email side effects.
